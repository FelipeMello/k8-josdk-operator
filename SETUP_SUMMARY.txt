PROJECT ARCHITECTURE
================================================================================
Spring Boot Initialization
  ↓
Kubernetes Client Creation (KubernetesClientBuilder)
  ↓
Operator Creation (with custom client)
  ↓
Reconciler Registration (WebPageReconciler)
  ↓
Operator.start() - Watches for WebPage resources
  ↓
When WebPage resource changes → WebPageReconciler.reconcile() is called
================================================================================
WHY SPRING BOOT + JOSDK?
================================================================================
✅ Recommended approach for production Kubernetes operators
Advantages:
  • Dependency Injection - Spring manages bean lifecycle
  • Configuration Management - Use application.yml for settings
  • Monitoring - Built-in actuators (/actuator/health, /actuator/metrics)
  • Structured Logging - SLF4J + Logback integration
  • REST APIs - Easy to add HTTP endpoints
  • Testing Framework - Spring Boot test utilities
  • External Services - Simple integration with other services
Trade-offs (acceptable):
  • Larger JAR size (~37MB vs ~10MB for pure JOSDK)
  • Longer startup time (~2-3 seconds vs ~500ms)
  • Slightly higher memory footprint
